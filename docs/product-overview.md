# プロダクト概要

## コンセプト

課題曲と担当パートを公開し、参加者のエントリーから演奏成立までを一元管理するサービス。

一般的なイベント募集サービスとの違いは、参加者を集めるだけでなく、曲ごとの編成を成立させ、当日の演奏準備まで支援する点にある。

## 最初の対象ユーザー

Googleフォーム、表計算、SNS、メッセージアプリなどを組み合わせて、課題曲型の音楽セッションを運営している主催者を主対象とする。

### 主催者の課題

- 誰がどの曲・パートに応募したか把握しづらい
- 不足パートや未成立曲を確認しづらい
- 同じパートに希望者が重複した際の調整が大変
- キャンセル発生後の再募集が面倒
- 譜面、参考音源、演奏上の連絡が複数箇所に散らばる

### 参加者の課題

- 自分が応募できる曲やパートを探しづらい
- 応募中、保留、担当確定などの状態が分かりづらい
- 演奏キー、曲構成、参考資料の最新版を確認しづらい

## MVPで提供する価値

主催者が、次の一連の作業をひとつのサービス内で完結できるようにする。

1. セッションを作成する
2. 課題曲と募集パートを登録する
3. 参加申込を受け付けて承認する
4. 曲・パートへの応募を受け付ける
5. 応募者を調整して担当者を確定する
6. 曲の成立状況を把握する
7. セットリストと演奏資料を共有する
8. 参加者へお知らせを送る

## MVPの機能

### 主催者向け

- セッションの作成、編集、公開
- 開催概要、参加条件、募集期限の設定
- 課題曲と募集パートの登録
- 参加申込の承認、見送り
- 曲・パート応募の確認
- 担当者の確定、保留、見送り
- 曲ごとの成立状況の確認
- セットリストと共有資料の公開
- 参加者への一斉連絡
- 運営メンバーの追加

### 参加者向け

- 公開セッションの検索、閲覧
- 楽器、経験レベルなどのプロフィール登録
- イベントへの参加申込
- 希望曲、パート、希望順位の登録
- 申込、応募、担当確定状況の確認
- 参考音源、譜面、演奏メモの閲覧
- キャンセル申請

## MVP対象外

- サービス内決済
- 個別チャット
- ユーザー間の相互評価
- 動画投稿
- AIによる推薦
- フォロー、タイムライン
- ネイティブモバイルアプリ

## 成功の仮説

フォームと表計算で行われている募集・編成管理を置き換えられれば、主催者に継続利用される。特に「曲別編成表を見れば、応募、担当、不足パートが分かる」状態をMVPの中心価値とする。
